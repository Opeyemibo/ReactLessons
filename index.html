Environment Setup – Download & Configure React

  1. Install Node.js

    Visit nodejs.org and download the LTS (Long-Term Support) version. Run the installer and keep the default options. This installs Node.js and the npm package manager needed to run React projects.

    Check: Open your terminal and run node -v and npm -v. You should see version numbers.

  2. Choose an Editor

    Visual Studio Code is a great free option. Download it from code.visualstudio.com and install the JavaScript, React, and ESLint extensions for a smoother experience.

  3. Create a New React Project

    Option A: Vite (fast modern tooling)

    1. Run npm create vite@latest react-playbook -- --template react.
    2. cd react-playbook
    3. npm install
    4. npm run dev and open the shown local URL in the browser.

    Option B: Create React App (friendly classic experience)

    1. npx create-react-app react-playbook
    2. cd react-playbook
    3. npm start to launch the development server.

    Both options install React, ReactDOM, and supporting tooling. Pick one and stick with it through the lessons.

  4. Optional: Configure Git

    Install Git, then run git init inside your project folder to start saving checkpoints of your progress.

Your environment is ready once you can run npm run dev or npm start and see the React starter page in your browser.

Module 1 · JavaScript Foundations

These lessons build a solid JavaScript base so you can read and write React code with confidence.

JavaScript is the language that powers interactivity on the web. You can run it directly in any browser or on your machine with Node.js. Learn how to embed a simple script in an HTML file and how to open the browser console to experiment.

Practical Exercise

  1. Create hello.html with a script that logs "Hello, JavaScript".
  2. Open it in your browser and use the console to change the message.
  3. Try alert() to pop up a greeting message.

JavaScript stores information in variables created with let, const, or var. Primitive types include strings, numbers, booleans, null, and undefined. Objects and arrays group related data.

Practical Exercise

  1. Declare variables for a learner's name, enrolled status, and progress percentage.
  2. Create an object student that combines those values.
  3. Use console.table(student) to inspect the object.

Functions package reusable logic. Learn declaration vs. arrow functions and how scope determines where variables can be used. Understand parameters, return values, and default arguments.

Practical Exercise

  1. Write a function greetStudent(name = "Coder") that returns a greeting string.
  2. Create an arrow function calculateScore(completedLessons, totalLessons) returning a percentage.
  3. Log both results to reinforce function usage.

Arrays hold ordered lists while objects store key/value pairs. Discover methods like push, map, and filter that you will use frequently in React to transform data.

Practical Exercise

  1. Create an array of lesson titles.
  2. Use map to create a numbered list of titles.
  3. Filter lessons that include the word "React" and log the result.

The Document Object Model (DOM) represents HTML as objects. Learn how to query elements, update text, change classes, and respond to user events. These skills translate directly to how React manages the UI for you.

Practical Exercise

  1. Create a simple HTML list of lessons.
  2. Use JavaScript to append a new lesson item when a button is clicked.
  3. Toggle a CSS class on click to highlight completed lessons.

Mini Project · Learning Journal

Build a small web page that lists the lessons you have completed. Include buttons to mark lessons as done and store the count in localStorage so progress persists.

Module 2 · Modern JavaScript for React

Upgrade your toolkit with ES6+ features and tooling patterns used in every React project.

Explore template literals, destructuring, spread/rest operators, and shorthand object properties. These patterns keep React components clean and readable.

Practical Exercise

  1. Destructure fields from a student object.
  2. Create arrays using spread syntax to add new lessons without mutating existing ones.
  3. Write a function that uses rest parameters to accept any number of scores.

React apps frequently fetch data. Learn how promises, fetch, async, and await make asynchronous code easy to read.

Practical Exercise

  1. Use fetch to request mock lesson data from JSONPlaceholder.
  2. Show a loading message while waiting for the response.
  3. Handle errors with try...catch.

Modern projects break code into modules. Learn named vs. default exports, how bundlers assemble files, and how to structure folders for clarity.

Practical Exercise

  1. Create mathTools.js exporting helper functions.
  2. Import them into another file and log results.
  3. Move your HTML/JS into a simple Vite project to experience module imports.

Project · Course Planner Dashboard

Build a dashboard that loads lesson data from a JSON file (or API) and displays status filters (All, In Progress, Completed). This prepares you for React state and props.

Module 3 · React Fundamentals

Learn what React is, why it matters, and how to build your first components.

JSX lets you write HTML-like syntax in JavaScript. Under the hood it calls React.createElement. Learn about components, props, and the virtual DOM.

Practical Exercise

  1. Create a new React component Header that returns a <header> element.
  2. Render it inside App.jsx.
  3. Pass a title prop and display it.

Components communicate through props. Parent components pass data while children display or transform it. Treat props as read-only.

Practical Exercise

  1. Create LessonCard that accepts title, summary, and level.
  2. Render multiple cards from an array using map.
  3. Add PropTypes (or TypeScript types) to document the props.

Use event handlers to respond to clicks and form submissions. Learn how to prevent default browser behavior and how to pass data into handlers.

Practical Exercise

  1. Add a button that logs "Lesson started" when clicked.
  2. Build a controlled input that updates state as you type.
  3. Handle form submission, preventing reload with event.preventDefault().

Project · React Lesson Library

Build a React app that lists lessons with title, difficulty, and a "Start" button for each. Use props to pass data to reusable components and track what the learner has opened.

Module 4 · State, Effects & Data Flow

Master how React tracks changes, reacts to user input, and talks to external APIs.

useState lets components remember values between renders. Learn how to update state immutably and how React re-renders with new state.

Practical Exercise

  1. Create a progress tracker with useState storing completed lesson count.
  2. Add increment/decrement buttons with safeguards against negative values.
  3. Display a congratulatory message when all lessons are complete.

useEffect runs code after a render. Use it for data fetching, subscriptions, and syncing state with the outside world. Learn dependency arrays and cleanup functions.

Practical Exercise

  1. Fetch lesson data from a public API and show a loading indicator.
  2. Store fetched data in state and render it.
  3. Clean up a timer or event listener inside the cleanup function.

Controlled components keep the source of truth in React state. Manage form inputs, validation, and submit handlers to capture user data cleanly.

Practical Exercise

  1. Build a form to collect lesson feedback with name, rating, and comments.
  2. Validate that name and rating are provided.
  3. Display submitted feedback below the form.

When multiple components need the same data, lift state to their closest common parent. Learn patterns for passing down state and updater functions via props.

Practical Exercise

  1. Create ModuleList with child ModuleFilter and ModuleCards.
  2. Store filter state in the parent and pass setters to the filter.
  3. Render filtered modules in the cards component.

Project · React Progress Tracker

Combine everything to track progress across modules. Fetch module data, filter lessons, and allow learners to mark items complete. Persist data in localStorage or a lightweight backend.

Module 5 · Architecture, Routing & Beyond

Take your React apps from small demos to production-ready projects by organizing code and adding routing, context, and testing.

Discover common folder structures (src/components, src/hooks, etc.), CSS Modules, styled-components, and utility-first CSS frameworks like Tailwind.

Practical Exercise

  1. Refactor a project to group components by feature.
  2. Add global styles and component-specific styles.
  3. Document the structure in a README.

Use React Router to create multi-page experiences without leaving the SPA. Define routes, use navigation components, and render route parameters.

Practical Exercise

  1. Install React Router with npm install react-router-dom.
  2. Create routes for Home, Lessons, and Projects.
  3. Use useParams to load dynamic lesson details.

Context lets you share data without prop drilling. Learn when to use context, how to create providers, and alternatives like Redux and Zustand.

Practical Exercise

  1. Create a ProgressContext that stores lessons completed.
  2. Wrap your app with the provider.
  3. Consume the context in nested components to show global progress.

Testing ensures confidence. Learn how to write unit tests with Jest and React Testing Library, then deploy to Vercel, Netlify, or GitHub Pages.

Practical Exercise

  1. Write a test that checks if your lesson cards render the correct title.
  2. Use @testing-library/react to simulate a button click.
  3. Deploy your app with npm run build and connect to a hosting provider.

Capstone Project · React Learning Portal

Design a multi-page React application that features lesson navigation, progress tracking, feedback forms, and a personalized dashboard. Include routing, context for global state, and at least one API integration.

Project Ideas & Milestones

  • Project 1 · JS Quiz App

    Create a multiple-choice quiz that tests JavaScript basics. Show score feedback and allow retakes.

  • Project 2 · React Study Planner

    Use React to plan weekly study sessions, mark tasks complete, and display motivational quotes fetched from an API.

  • Project 3 · Component Library

    Build a reusable set of buttons, cards, and navigation components. Document them in Storybook or a custom showcase page.

  • Project 4 · Full-stack Learning Tracker

    Combine React with a backend (Firebase, Supabase, or Express) to persist lessons, achievements, and notes.

Learning Strategies & Study Tips

  • Chunk learning: Focus on one lesson per session and complete the exercise before moving on.
  • Teach back: Explain concepts aloud or in a journal—teaching helps you remember.
  • Practice daily: Short sessions every day beat one long session each week.
  • Reflect: After each project, note what went well and what you would improve.
